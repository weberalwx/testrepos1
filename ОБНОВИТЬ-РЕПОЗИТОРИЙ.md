# Как заменить старую версию на GitHub на новую

У вас уже есть репозиторий на GitHub со старой версией. Нужно отправить туда новую версию из папки `C:\Users\gusev\Desktop\minesweeper` (с фоном, новым дизайном, Z вместо бомб).

---

## Шаг 1. Узнать URL вашего репозитория

Зайдите на GitHub в свой репозиторий с игрой. Нажмите зелёную кнопку **«Code»** и скопируйте URL — он будет вида:

`https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git`

(Например: `https://github.com/ivanov/minesweeper.git`)

---

## Шаг 2. Настроить Git (если ещё не делали)

Откройте **PowerShell** или **Командную строку** и выполните один раз (подставьте свой email и имя):

```powershell
git config --global user.email "ваш@email.com"
git config --global user.name "Ваше Имя"
```

---

## Шаг 3. Привязать папку к репозиторию и отправить новую версию

Выполните по очереди (подставьте **свой** URL из шага 1):

```powershell
cd C:\Users\gusev\Desktop\minesweeper
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
git add .
git commit -m "Новая версия: фон, обновлённый дизайн, Z вместо бомб"
git branch -M main
git push -u origin main --force
```

**Важно:** `--force` перезапишет историю на GitHub и заменит старую версию на новую. Если репозиторий один только ваш — это нормально.

Если Git напишет, что `remote origin already exists`, тогда сначала удалите старую привязку и добавьте заново:

```powershell
git remote remove origin
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
git push -u origin main --force
```

---

## Готово

После `git push` на GitHub будет новая версия: обновлённый `index.html`, файл `background.png` (фон), скрипты и инструкции. Старая версия в истории коммитов заменится на новую.

Если у вас включён **GitHub Pages** (Settings → Pages → Source: main), игра через 1–2 минуты откроется по ссылке с новым фоном и дизайном.
